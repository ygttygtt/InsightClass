## 3. 基本用法

### 3.1 按视角分组录制

```bash
# 录制全部前视角（7路）
python scripts/record_multi_rtsp.py --view front

# 录制全部后视角（9路）
python scripts/record_multi_rtsp.py --view rear
```

### 3.2 只录部分IP（组内分工）

每人电脑性能不同，可以只录自己负责的几路：

```bash
# 前视角中只录3路
python scripts/record_multi_rtsp.py --view front --ips 10.8.14.36 10.8.14.34 10.8.14.30

# 后视角中只录2路
python scripts/record_multi_rtsp.py --view rear --ips 10.8.14.5 10.8.14.29
```

`--view` 决定文件保存到哪个目录，`--ips` 决定具体录哪些摄像头。

### 3.3 不区分视角

```bash
python scripts/record_multi_rtsp.py --ips 10.8.14.36 10.8.14.34 10.8.14.5
```

文件保存到 `data/raw_videos/manual/`。

## 4. 输出文件

```
data/raw_videos/
├── front/          # --view front 的输出
│   ├── 10.8.14.36_20260528_143000.mp4
│   ├── 10.8.14.34_20260528_143000.mp4
│   └── ...
├── rear/           # --view rear 的输出
│   ├── 10.8.14.5_20260528_143500.mp4
│   └── ...
└── manual/         # 只用 --ips（不指定--view）的输出
```

文件名格式：`{IP}_{录制开始时间}.mp4`

## 5. 停止录制

按 **Ctrl + C** 即可停止。每路视频会正常关闭并保存，不会损坏文件。

## 6. 录制参数调整

如需调整帧率、分辨率，编辑 `scripts/record_multi_rtsp.py` 顶部的配置：

```python
RECORD_FPS = 15          # 录制帧率，15 对行为检测够用，减小文件体积
RECORD_WIDTH = None      # None = 保持原始分辨率；设数字则缩放
FOURCC = "avc1"          # H.264 编码，不兼容时自动回退 mp4v
```

## 7. 预览模式（调试用）

如果录制前想确认画面是否正常，加 `--preview`：

```bash
python scripts/record_multi_rtsp.py --view front --ips 10.8.14.36 --preview
```

每个摄像头会弹出一个640x360的预览窗口。确认画面正常后，去掉 `--preview` 重新开始正式录制。

## 8. 常见问题

### Q1: 连接失败

```
[10.8.14.xx] ❌ 连接失败，跳过
```

检查：
1. 电脑是否能 ping 通该 IP
2. 摄像头是否在线
3. 用户名密码是否正确

### Q2: 视频流中断

脚本会自动重连。如果反复中断，可能是网络不稳定，可尝试只录更少的IP。

### Q3: 电脑性能不够

减少同时录制的路数。比如前视角7路由3个人分工，每人录2-3路：

```bash
# A同学
python scripts/record_multi_rtsp.py --view front --ips 10.8.14.36 10.8.14.34
# B同学
python scripts/record_multi_rtsp.py --view front --ips 10.8.14.30 10.8.14.10
# C同学
python scripts/record_multi_rtsp.py --view front --ips 10.8.14.18 10.8.14.26 10.8.14.28
```

文件都在 `data/raw_videos/front/` 下，最后汇总到一起即可。

### Q4: 录制文件打不开或只有声音没有画面

可能编码不兼容。把脚本顶部的 `FOURCC` 改为 `"mp4v"`，这是最通用的编码。
