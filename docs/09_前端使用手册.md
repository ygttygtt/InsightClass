# InsightClass 前端使用手册

> 本文档介绍如何使用 InsightClass 深见课堂的 Web 前端界面进行实时行为检测。

---

## 一、启动服务

在项目根目录执行：

```bash
insightclass serve --host 127.0.0.1 --port 8000
```

然后在浏览器中打开 **http://127.0.0.1:8000**。

如果需要让局域网内其他设备访问，使用 `--host 0.0.0.0`：

```bash
insightclass serve --host 0.0.0.0 --port 8000
```

## 四、功能详解

### 4.1 RTSP 监控（主功能）

这是系统的核心功能，连接教室前排摄像头进行实时行为检测。

**操作步骤：**

1. 点击左侧「RTSP 监控」
2. 在摄像头列表中选择一个摄像头（IP 地址）
3. 点击顶部「开始」按钮
4. 系统会不断从摄像头抓取画面，每帧运行 AI 检测
5. 检测结果以彩色框叠加在画面上：
   - 红色：玩手机
   - 蓝色：交谈
   - 黄色：打瞌睡
6. 底部统计栏实时显示各类别检测数量和推理延迟
7. 点击「停止」结束检测

**摄像头分组：**

| 分组 | 数量 | 说明 |
|------|------|------|
| 前视角 | 7 个 | 教室前排摄像头，覆盖学生正面 |
| 后视角 | 9 个 | 教室后排摄像头，覆盖学生背面 |

**注意事项：**

- RTSP 检测需要在同一局域网内
- 每帧检测耗时取决于硬件（CPU/GPU），通常 100-500ms
- 如果某个摄像头连接失败，画面会显示黑屏，可切换到其他摄像头

### 4.2 电脑摄像头

使用笔记本电脑的内置摄像头进行检测。适合在本地快速测试模型效果。

**操作步骤：**

1. 点击左侧「电脑摄像头」
2. 点击顶部「开始」
3. 浏览器会请求摄像头权限，点击「允许」
4. 检测会自动开始，实时叠加检测框
5. 点击「停止」结束

**注意事项：**

- 需要浏览器授予摄像头权限
- 笔记本摄像头视野有限，适合测试，不适合正式课堂监控
- 检测效果取决于摄像头角度和光线条件

### 4.3 图片检测

上传一张图片，系统会在图片上标注检测到的行为。

**操作步骤：**

1. 点击左侧「图片检测」
2. 点击上传区域，或直接拖拽图片到区域中
3. 支持 JPG、PNG 格式
4. 点击「开始检测」
5. 等待分析完成，结果直接显示在画面上

**适用场景：**

- 快速测试单张图片的检测效果
- 检查模型在特定场景下的识别能力
- 演示和汇报

### 4.4 视频检测

上传一个视频文件，系统逐帧分析后播放检测结果。

**操作步骤：**

1. 点击左侧「视频检测」
2. 点击上传区域，或直接拖拽视频到区域中
3. 支持 MP4、AVI、MOV 格式
4. 点击「开始分析」
5. 等待服务端完成全部帧的推理（可能需要几十秒到几分钟）
6. 分析完成后，视频自动播放，检测框实时叠加

**注意事项：**

- 视频越长、分辨率越高，分析时间越长
- 分析期间会显示加载提示，请耐心等待
- 分析完成后可以像普通视频一样暂停、拖动进度条

## 三、模型与参数

### 3.1 模型选择

下拉菜单列出所有已训练的实验模型。选择不同的模型会使用对应的权重文件进行检测。

- 如果只有一个模型，无需切换
- 如果有多个模型，可以通过对比选择效果最好的

### 3.2 置信度阈值

控制检测的"严格程度"：

| 值 | 效果 |
|----|------|
| 低（0.05-0.20） | 更多检测结果，可能有误报 |
| 中（0.25-0.45） | 平衡精度和召回率（推荐） |
| 高（0.50-0.95） | 更少检测结果，只显示高置信度的 |

### 3.3 IoU 阈值

控制重叠检测框的合并程度：

| 值 | 效果 |
|----|------|
| 低（0.1-0.3） | 更积极合并重叠框 |
| 中（0.4-0.5） | 标准合并（推荐） |
| 高（0.6-0.9） | 保留更多独立框 |

## 五、常见问题

### Q: 页面打不开？

- 确认服务已启动（终端没有报错）
- 确认端口没被占用
- 检查浏览器地址是否正确

### Q: RTSP 摄像头连不上？

- 确认电脑和摄像头在同一局域网
- 尝试在终端 ping 摄像头 IP
- 检查摄像头是否在线（用监控软件确认）

### Q: 检测延迟很高？

- CPU 推理较慢是正常的，通常 200-500ms/帧
- 如果有 NVIDIA GPU，安装 CUDA 版 PyTorch 可大幅加速
- 降低图片分辨率（在训练时使用更小的 imgsz）

### Q: 检测结果不准？

- 调整置信度阈值（降低可增加召回，升高可减少误报）
- 检查摄像头角度是否能看到学生上半身
- 确认训练数据质量（标注是否准确、数据量是否足够）

### Q: 视频上传分析卡住？

- 视频文件过大会导致分析时间较长
- 检查终端是否有错误输出
- 尝试使用更短的视频片段
