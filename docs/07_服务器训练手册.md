# 服务器训练手册（V2 数据集）

本手册指导你在 GPU 服务器上完成 InsightClass V2 数据集的模型训练。

## 前置条件

- 服务器已安装 Anaconda/Miniconda
- 服务器已安装 NVIDIA 驱动和 CUDA
- 你有服务器的 SSH 访问权限

## 1. 本地准备

### 1.1 确认 V2 数据集完整

在本地检查数据集状态：

```bash
# 检查数据集结构
ls -la data/processed/classroom_behavior_v2/

# 预期输出：
# images/
#   train/  (约 989 张)
#   val/    (约 175 张)
#   test/   (约 327 张)
# labels/
#   train/
#   val/
#   test/
# dataset.yaml
# manifest.yaml
# frame_index.csv
```

### 1.2 打包 V2 数据集

**注意：图片文件较大，不上传到 Gitee，直接传输到服务器。**

```bash
# 在本地项目根目录执行
cd data/processed/

# 打包 classroom_behavior_v2 文件夹
tar -czf classroom_behavior_v2.tar.gz classroom_behavior_v2/

# 检查打包文件大小
ls -lh classroom_behavior_v2.tar.gz
```

### 1.3 推送代码到 Gitee

```bash
# 回到项目根目录
cd ../../

# 添加配置文件和文档（数据集文件不上传，通过 scp 传输）
git add configs/classes.yaml
git add configs/training_v2_ultralytics.yaml
git add docs/07_服务器训练手册.md

# 提交
git commit -m "feat: add V2 dataset config and training manual"

# 推送
git push origin main
```

**注意：** `data/processed/`、`reports/`、`experiments/` 目录已被 `.gitignore` 忽略，不需要也不应该添加到 git。

## 2. 服务器操作

### 2.1 SSH 登录服务器

```bash
ssh your_username@server_ip
```

### 2.2 克隆代码

```bash
# 进入工作目录
cd /home/your_username/

# 克隆项目
git clone https://gitee.com/your_username/InsightClass.git

# 进入项目目录
cd InsightClass
```

### 2.3 上传数据集

**方法一：使用 scp（推荐）**

```bash
# 在本地终端执行（不是服务器）
scp data/processed/classroom_behavior_v2.tar.gz your_username@server_ip:/home/your_username/InsightClass/data/processed/
```

**方法二：使用 SFTP 工具**

使用 FileZilla、WinSCP 等工具，将 `classroom_behavior_v2.tar.gz` 上传到服务器的 `InsightClass/data/processed/` 目录。

### 2.4 解压数据集

```bash
# 在服务器上执行
cd /home/your_username/InsightClass/data/processed/

# 解压
tar -xzf classroom_behavior_v2.tar.gz

# 验证解压结果
ls -la classroom_behavior_v2/

# 删除压缩包（可选，节省空间）
rm classroom_behavior_v2.tar.gz
```

### 2.5 创建 Conda 环境

```bash
# 回到项目根目录
cd /home/your_username/InsightClass/

# 创建环境
conda create -n insightclass python=3.10 -y

# 激活环境
conda activate insightclass

# 安装项目依赖
pip install -e .[ultralytics,dev]
```

### 2.6 验证环境

```bash
# 检查 PyTorch 和 CUDA
python -c "import torch; print(f'PyTorch: {torch.__version__}'); print(f'CUDA available: {torch.cuda.is_available()}'); print(f'CUDA version: {torch.version.cuda}')"

# 检查 ultralytics
python -c "import ultralytics; print(f'Ultralytics: {ultralytics.__version__}')"

# 运行环境检查脚本（如果存在）
python test_environment.py
```

**预期输出：**
```
PyTorch: 2.x.x+cu121
CUDA available: True
CUDA version: 12.x
Ultralytics: 8.x.x
```

### 2.7 重新生成数据集配置

**重要：** `dataset.yaml` 中的路径是 Windows 本地路径，在 Linux 服务器上需要重新生成。

```bash
# 重新生成 dataset.yaml（路径会自动适配服务器环境）
python -m insightclass write-yolo-yaml \
  --dataset-root data/processed/classroom_behavior_v2 \
  --class-config configs/classes.yaml \
  --output data/processed/classroom_behavior_v2/dataset.yaml

# 验证生成的配置
cat data/processed/classroom_behavior_v2/dataset.yaml

# 运行数据集检查
python -m insightclass inspect-yolo \
  --dataset-root data/processed/classroom_behavior_v2 \
  --class-config configs/classes.yaml \
  --output reports/v2_server_inspection.json

# 查看检查报告
cat reports/v2_server_inspection.json
```

**预期结果：**
- train: ~989 张图片
- val: ~175 张图片
- test: ~327 张图片
- 无 missing_label_files、empty_label_files、invalid_class_ids 等问题

## 3. 开始训练

### 3.1 检查训练配置

```bash
cat configs/training_v2_ultralytics.yaml
```

**配置说明：**

| 参数 | 值 | 说明 |
|------|-----|------|
| backend | ultralytics | 使用 Ultralytics YOLO |
| data_config_path | data/processed/classroom_behavior_v2/dataset.yaml | V2 数据集配置 |
| model_weights | yolo11n.pt | 预训练模型（首次运行自动下载） |
| image_size | 960 | 输入图片尺寸 |
| epochs | 80 | 训练轮次 |
| batch_size | 16 | 批次大小（根据 GPU 显存调整） |
| device | 0 | GPU 编号 |
| patience | 30 | 早停耐心值 |

### 3.2 启动训练

```bash
# 确保在项目根目录，且 conda 环境已激活
conda activate insightclass

# 开始训练
python -m insightclass train --config configs/training_v2_ultralytics.yaml
```

**训练过程：**
1. 自动下载 `yolo11n.pt` 预训练模型（首次）
2. 加载 V2 数据集
3. 开始训练，显示进度条和指标
4. 训练完成后保存结果到 `experiments/` 目录

### 3.3 监控训练（可选）

```bash
# 在另一个终端窗口查看 GPU 使用情况
watch -n 1 nvidia-smi

# 查看训练日志
tail -f experiments/baseline_yolo11n_v2_e80/train.log
```

### 3.4 训练时间预估

根据 GPU 性能不同，训练时间参考：

| GPU | 预计时间 |
|-----|---------|
| RTX 3090/4090 | 30-60 分钟 |
| RTX 3080 | 60-90 分钟 |
| V100 | 45-75 分钟 |
| A100 | 20-40 分钟 |

## 5. 故障排除

### 5.1 CUDA 内存不足 (OOM)

**症状：** `RuntimeError: CUDA out of memory`

**解决方案：**
```bash
# 方案1：减小 batch_size
# 编辑 configs/training_v2_ultralytics.yaml
# 将 batch_size: 16 改为 batch_size: 8 或 batch_size: 4

# 方案2：减小 image_size
# 将 image_size: 960 改为 image_size: 640
```

### 5.2 数据集路径错误

**症状：** `FileNotFoundError` 或 `No such file`

**解决方案：**
```bash
# 检查 dataset.yaml 中的路径
cat data/processed/classroom_behavior_v2/dataset.yaml

# 确认 path 字段指向正确目录
# 如果路径不对，重新生成：
python -m insightclass write-yolo-yaml \
  --dataset-root data/processed/classroom_behavior_v2 \
  --class-config configs/classes.yaml \
  --output data/processed/classroom_behavior_v2/dataset.yaml
```

### 5.3 依赖版本冲突

**症状：** `ImportError` 或版本不兼容

**解决方案：**
```bash
# 重新安装依赖
pip install -e .[ultralytics,dev] --force-reinstall
```

### 5.4 训练中断恢复

**症状：** 训练意外中断

**解决方案：**
```bash
# 使用 resume 参数（如果支持）
python -m insightclass train --config configs/training_v2_ultralytics.yaml --resume

# 或者从 last.pt 继续训练
# 修改 training_v2_ultralytics.yaml 中的 model_weights 为 last.pt 的路径
```
