# 服务器训练手册（V2 数据集）

本手册指导你在 GPU 服务器上完成 InsightClass V2 数据集的模型训练。

## 前置条件

- 服务器已安装 Python 3.10+ 和 pip
- 服务器已安装 NVIDIA 驱动和 CUDA
- 你有服务器的 SSH 访问权限
- 模型权重文件 (`yolo11n.pt`, `yolo26n.pt`) 已随项目 git 仓库分发

## 服务器环境

| 项目 | 值 |
|------|-----|
| 用户名 | `lb` |
| 工作目录 | `/home/lb/GXQ/InsightClass` |
| Python | 3.10.12 (`python3`) |
| GPU | 4x NVIDIA A100-PCIE-40GB |
| CUDA | 12.8 |
| OS | Ubuntu 22.04 |
| 包管理 | venv（无 conda） |

## 1. 本地准备（Windows 笔记本）

### 1.1 打包数据集

```bash
cd data/processed/
tar -czf classroom_behavior_v2.tar.gz classroom_behavior_v2/
ls -lh classroom_behavior_v2.tar.gz
cd ../../
```

### 1.2 推送代码到 Gitee

```bash
git add configs/classes.yaml
git add configs/training_v2_ultralytics.yaml
git add configs/training_v2_ultralytics_yolo26n.yaml
git add docs/07_服务器训练手册.md
git add scripts/
git add yolo11n.pt yolo26n.pt
git commit -m "feat: V2 dataset config, training scripts and pretrained weights"
git push origin main
```

### 1.3 上传数据集到服务器

```bash
# 方法一：scp（在本地终端执行）
scp data/processed/classroom_behavior_v2.tar.gz lb@服务器IP:/home/lb/GXQ/InsightClass/data/processed/

# 方法二：WinSCP / FileZilla 拖拽上传到 /home/lb/GXQ/InsightClass/data/processed/
```

---

## 2. 服务器操作（SSH 登录后）

### 2.1 拉取代码 + 解压数据集

```bash
cd /home/lb/GXQ/InsightClass

# 拉取最新代码（包含脚本和模型权重）
git pull

# 解压数据集
cd data/processed/
tar -xzf classroom_behavior_v2.tar.gz
rm classroom_behavior_v2.tar.gz
cd ../..
```

### 2.2 一键配置环境

```bash
bash scripts/server_setup.sh
```

这一条命令自动完成：
- 创建 Python venv（`.venv/`）
- 安装所有依赖（ultralytics、torch 等）
- 生成 `dataset.yaml`（适配 Linux 路径）
- 运行数据集质量检查

### 2.3 开始训练

```bash
# 训练 yolo11n（前台运行）
bash scripts/server_train.sh yolo11n

# 训练 yolo26n
bash scripts/server_train.sh yolo26n

# 后台运行（推荐，SSH 断开不影响）
bash scripts/server_train.sh yolo11n bg
```

后台运行时，脚本会输出日志文件名和 PID：
```
查看日志:  tail -f train_yolo11n_20260529_143000.log
查看 GPU:  watch -n 1 nvidia-smi
停止训练:  kill <PID>
```

## 4. 故障排除

| 问题 | 解决 |
|------|------|
| `CUDA out of memory` | 编辑配置文件，`batch_size` 改为 8 或 4 |
| `FileNotFoundError` dataset.yaml | 重新运行 `bash scripts/server_setup.sh` |
| `pip install` 超时 | 网络问题，重试或换源：`pip install -i https://pypi.tuna.tsinghua.edu.cn/simple ...` |
| 训练中断 | 重新运行 `bash scripts/server_train.sh`，ultralytics 会自动从 `last.pt` 恢复 |
| SSH 断开导致训练停止 | 使用 `bash scripts/server_train.sh yolo11n bg` 后台运行 |

## 5. 高级：多 GPU 训练

编辑 `configs/training_v2_ultralytics.yaml`，将 `device` 改为：

```yaml
device: "0,1,2,3"  # 使用全部 4 张 A100
```

A100-40GB 显存充足，`batch_size: 16` 可以适当调大到 32。
