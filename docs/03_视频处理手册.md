# 视频处理手册 — 从录制视频到可标注图片

> 录制完成后，将原始视频抽帧为图片，供标注使用。

## 1. 前置条件

- 录制已结束，视频文件位于 `data/raw_videos/front/`（每人录制的 IP 不同，汇总后统一处理）
- 已安装项目依赖：`pip install -e .[ultralytics]`
- 确认 opencv 可用：`python -c "import cv2; print(cv2.__version__)"`

## 2. 整体流程

```
data/raw_videos/front/*.mp4
        │
        ▼
  create-manifest    ← 扫描视频，按视频级切分 train/val/test
        │
        ▼
  manifest.yaml      ← 记录视频列表、切分方案、元信息
        │
        ▼
  extract-frames     ← 按指定 fps 抽帧，输出图片 + 帧索引
        │
        ▼
data/processed/classroom_behavior_v2/
  ├── images/{train,val,test}/*.jpg
  └── frame_index.csv
        │
        ▼
  标注（见 04_X-AnyLabeling操作手册.md）
```

## 3. Step 1 — 创建 Manifest

Manifest 记录了所有视频的列表、train/val/test 切分方案和元信息。切分是按视频级别的，同一视频的帧不会泄漏到不同 split。

```bash
python -m insightclass create-manifest \
  --config configs/dataset_manifest_v2.yaml \
  --output data/processed/classroom_behavior_v2/manifest.yaml
```

执行后会输出类似：

```
Saved manifest to /path/to/data/processed/classroom_behavior_v2/manifest.yaml
```

### 配置说明

`configs/dataset_manifest_v2.yaml` 中的关键参数：

| 参数 | 值 | 说明 |
|------|----|------|
| `raw_videos_dir` | `data/raw_videos/front` | 扫描此目录下的所有 .mp4 文件（递归） |
| `processed_dir` | `data/processed/classroom_behavior_v2` | 抽帧图片和元数据的输出目录 |
| `train_ratio` | `0.7` | 70% 视频用于训练 |
| `val_ratio` | `0.2` | 20% 视频用于验证 |
| `seed` | `42` | 随机种子，保证可复现 |

切分结果会写入 manifest.yaml，之后的抽帧步骤按此执行。

### 汇总多人录制

如果团队成员各自录制了不同 IP 的视频，汇总到同一个 `data/raw_videos/front/` 目录后再执行此步骤。脚本会递归扫描子目录，文件名格式 `{IP}_{时间戳}.mp4`，不同 IP 不会冲突。

## 4. Step 2 — 抽帧

```bash
python -m insightclass extract-frames \
  --manifest data/processed/classroom_behavior_v2/manifest.yaml \
  --fps 0.167 \
  --max-frames-per-video 300 \
  --target-width 960
```

### 参数说明

| 参数 | 值 | 说明 |
|------|----|------|
| `--fps` | `0.167` | 每 6 秒抽 1 帧，约 10 帧/分钟 |
| `--max-frames-per-video` | `300` | 每个视频最多 300 帧（约 30 分钟内容） |
| `--target-width` | `960` | 缩放到 960px 宽，与录制和训练分辨率对齐 |
| `--seed` | `42` | 随机种子（默认值） |

### 抽帧逻辑

- 按 `native_fps / target_fps` 计算间隔，均匀采样
- 如果候选帧数超过 `max_frames_per-video`，随机抽样（seed 固定，可复现）
- 每帧单独 seek 读取，不会解码整个视频

执行后会输出每个视频的抽帧数量：

```
[train] 10.8.14.36_20260528_143000.mp4: 180 frames extracted (sampled from 27000 total)
[train] 10.8.14.34_20260528_143000.mp4: 165 frames extracted (sampled from 24750 total)
[val] 10.8.14.30_20260528_143000.mp4: 150 frames extracted (sampled from 22500 total)
...
Saved frame index to /path/to/frame_index.csv
```

## 5. 输出目录结构

```
data/processed/classroom_behavior_v2/
├── manifest.yaml           ← 视频清单和切分方案
├── frame_index.csv         ← 每张图片的来源视频、帧号、时间戳
├── images/
│   ├── train/              ← 训练集图片
│   │   ├── 10.8.14.36_20260528_143000_f000090.jpg
│   │   ├── 10.8.14.36_20260528_143000_f000630.jpg
│   │   └── ...
│   ├── val/                ← 验证集图片
│   └── test/               ← 测试集图片
└── labels/                 ← 标注后才会有（下一步）
    ├── train/
    ├── val/
    └── test/
```

### 命名规范

图片文件名格式：`{视频名}_f{帧号:06d}.jpg`

- 视频名 = 原始文件名去掉扩展名（如 `10.8.14.36_20260528_143000`）
- 帧号 = 视频中的零基帧序号，6 位零填充
- 例：`10.8.14.36_20260528_143000_f000090.jpg` 表示该视频的第 90 帧

### frame_index.csv 字段

| 字段 | 说明 |
|------|------|
| `split` | train / val / test |
| `video_id` | 原始视频文件名 |
| `source_path` | 视频绝对路径 |
| `frame_index` | 视频中的帧序号 |
| `timestamp_sec` | 帧在视频中的时间（秒） |
| `image_path` | 输出图片的绝对路径 |

```bash
# 检查各 split 的图片数量
echo "train: $(ls data/processed/classroom_behavior_v2/images/train/ | wc -l)"
echo "val:   $(ls data/processed/classroom_behavior_v2/images/val/ | wc -l)"

# 随机看几张图确认质量
ls data/processed/classroom_behavior_v2/images/train/ | head -5
```

确认无误后，进入标注阶段。

## 7. 下一步 — 标注

图片准备好后，使用 X-AnyLabeling 进行标注。详见 [04_X-AnyLabeling操作手册.md](04_X-AnyLabeling操作手册.md)。

标注的关键步骤：

1. 打开 X-AnyLabeling，加载 `images/train/` 目录
2. 使用 AI 辅助标注 + 人工修正
3. 导出 YOLO 格式标签到 `labels/train/`
4. 对 `val/` 和 `test/` 重复

标注完成后，回到项目根目录执行质检：

```bash
# 质检
python -m insightclass inspect-yolo \
  --dataset-root data/processed/classroom_behavior_v2 \
  --class-config configs/classes.yaml \
  --output reports/v2_inspection.json

# 生成 YOLO 数据集配置
python -m insightclass write-yolo-yaml \
  --dataset-root data/processed/classroom_behavior_v2 \
  --class-config configs/classes.yaml \
  --output data/processed/classroom_behavior_v2/yolo_dataset.yaml
```

## 8. 常见问题

### Q1: 抽帧太慢

每帧需要单独 seek，大视频会比较慢。可以通过降低 `--max-frames-per-video` 减少每视频的帧数。

### Q2: 磁盘空间不够

960px 宽的 JPG 约 100-200KB/张。100 个视频各 300 帧 ≈ 3-6GB。可以通过降低 `--fps` 或 `--max-frames-per-video` 减少总量。

### Q3: 想重新抽帧

直接删除 `data/processed/classroom_behavior_v2/` 目录，重新执行 Step 1 和 Step 2。manifest 的 seed 固定，切分方案不会变。

### Q4: manifest.yaml 中的视频路径不对

manifest 创建时会记录视频的绝对路径。如果移动了视频目录，需要重新执行 Step 1 生成新的 manifest。
