# 标注规范

## 1. 类别定义

训练标签建议固定使用英文 ID，文档和展示用中文说明：

| 训练类 ID | 中文含义 | 说明 |
| --- | --- | --- |
| `phone_use` | 玩手机 | 学生正在手持、查看、操作手机，且手机行为可被画面支持 |
| `talking` | 交谈 | 学生明显在和他人说话、侧头交流或嘴部动作明显指向交流 |
| `sleeping` | 打瞌睡 | 学生闭眼、趴桌、低头持续失神等明显困倦状态 |

## 2. 正例、反例、模糊例

### `phone_use`

- 正例：
  - 手机完整可见，学生目光或手部操作明显指向手机。
  - 手机部分可见，但手持和注视关系清楚。
- 反例：
  - 仅拿着文具盒、纸张、计算器。
  - 桌面疑似手机，但无法确认属于当前学生正在使用。
- 模糊例：
  - 手机被遮挡一半，仅露出边角。
  - 学生低头，但看不清是否在看手机。
  - 处理原则：首批标注宁缺勿滥，不确定则不标，并单独收集为难例。

### `talking`

- 正例：
  - 两名或多名学生明显侧头交流，嘴部开启、表情互动明显。
  - 学生回头与后排交流，持续多个相邻帧可见。
- 反例：
  - 仅短暂转头、看黑板、看老师。
  - 嘴巴张开但处于打哈欠、笑、咳嗽等非交流状态。
- 模糊例：
  - 只有头部转动，没有口型和互动证据。
  - 处理原则：需要至少一个较强视觉证据，避免把普通转头都标成交谈。

### `sleeping`

- 正例：
  - 趴桌睡觉。
  - 长时间闭眼、头部明显下垂、持续性发困姿态。
- 反例：
  - 单纯低头写字。
  - 弯腰捡东西。
  - 看书导致头部下压，但精神状态正常。
- 模糊例：
  - 只看到侧脸闭眼一瞬间。
  - 头低但看不清是否清醒。
  - 处理原则：优先标注姿态稳定、持续明显的困倦状态。

## 3. 框选规则

- 首期统一采用“行为主体框”，即以学生人体上半身为主，必要时包含手机。
- 如果行为发生在手部和头部附近，优先保证头、手、手机同时落在框中。
- 框不要过松，不要故意把邻座学生大面积包进去。
- 框也不要过紧，避免只框手机或只框头部，导致行为语义丢失。
- 同一帧同一学生只保留最能代表当前行为的一个框。

## 4. 边界情况

### 遮挡

- 遮挡后仍能确定行为，可标。
- 遮挡过重、行为证据不足，不标。

### 多学生重叠

- 每个可分辨行为主体各自单独框选。
- 如果无法区分行为归属，不标。

### 手机半露出

- 手机半露出但行为关系明确，可标为 `phone_use`。
- 仅露出疑似手机边角、无法确认，不标。

### 侧脸闭眼

- 若姿态和上下文明显符合困倦，可标 `sleeping`。
- 如果只是正常眨眼或短时低头，不标。

## 5. 同帧多行为规则

- 首期默认同一学生同一帧只标一个主行为。
- 优先级建议：
  1. `phone_use`
  2. `sleeping`
  3. `talking`
- 例外情况：如果后续发现多标签场景很多，再升级为多任务或事件级标注，不在首期实现。

## 6. 标注质检建议

- 每轮标注后至少抽查 5% 图像。
- 每个类别都要看难例，尤其是后排小目标和遮挡场景。
- 维护一个“难例清单”，把争议样本路径、问题和处理结论记录下来。
