## 一、项目整体概览

### 这个项目是干什么的？

InsightClass 是一个**课堂学生行为检测系统**。简单来说，它能从前排摄像头拍到的教室画面中，自动识别出三种行为：

| 行为 ID | 中文名 | 场景举例 |
|---------|--------|----------|
| `phone_use` | 玩手机 | 学生低头看手机 |
| `talking` | 交谈 | 学生之间交头接耳 |
| `sleeping` | 打瞌睡 | 学生趴桌子或低头闭眼 |

### 整个项目分成哪几个阶段？

```
录制视频 → 抽帧 → 标注 → 训练模型 → 推理检测 → 可视化/Web 展示
```

你可以把它想象成"教电脑认人"的过程：

1. **拍视频**：用教室里的摄像头录制上课画面
2. **抽帧**：从视频里每隔几秒截一张图
3. **标注**：人工在图片上框出"这里有人在玩手机"、"这里有人在打瞌睡"
4. **训练**：把这些标注好的图片喂给 AI 模型，让它学会识别
5. **推理**：训练好的模型看到新图片就能自动识别
6. **展示**：把识别结果用网页或视频叠加的方式展示出来

### 技术栈（用了哪些工具？）

| 层次 | 技术 | 作用 |
|------|------|------|
| 编程语言 | Python 3.10+ | 整个项目的基础 |
| AI 模型 | Ultralytics YOLO (YOLO11n / YOLO26n) | 目标检测的核心引擎 |
| 图像处理 | OpenCV (cv2) | 读取视频、抽帧、图像处理 |
| 数据格式 | PyYAML、JSON、CSV | 配置文件和数据存储 |
| Web 框架 | FastAPI + Jinja2 + uvicorn | 网页端推理界面 |
| 可视化 | supervision | 在图片/视频上画检测框 |
| 包管理 | pip + setuptools (src-layout) | 安装和分发 |
| 版本控制 | Git + Gitee | 代码托管 |

## 三、核心业务流程

### 3.1 数据处理全流程（从视频到可训练的数据集）

```
┌─────────────┐     ┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  录制视频    │────▶│  创建清单    │────▶│  抽帧        │────▶│  人工标注    │
│             │     │              │     │              │     │              │
│ 多路摄像头  │     │ 扫描视频目录 │     │ 按 FPS 截图  │     │ 用标注工具   │
│ 同时录制    │     │ 按比例划分   │     │ 缩放到 960px │     │ 框出行为区域 │
│ 输出 .mp4   │     │ train/val/test│    │ 输出 .jpg    │     │ 输出 .txt    │
└─────────────┘     └──────────────┘     └──────────────┘     └──────────────┘
                                                                      │
                          ┌───────────────────────────────────────────┘
                          ▼
                ┌──────────────┐     ┌──────────────┐
                │  质量检查    │────▶│  生成 YAML   │
                │              │     │              │
                │ 检查标注文件 │     │ 生成 YOLO    │
                │ 是否完整/合法│     │ 训练所需配置 │
                └──────────────┘     └──────────────┘
```

**对应命令：**

```bash
# 第 1 步：创建数据集清单（划分 train/val/test）
insightclass create-manifest --config configs/dataset_manifest.example.yaml --output data/.../manifest.yaml

# 第 2 步：从视频中抽帧
insightclass extract-frames --manifest <manifest路径> --fps 1.0 --target-width 960

# 第 3 步：人工标注（用 X-AnyLabeling 等外部工具，非命令行）

# 第 4 步：检查标注质量
insightclass inspect-yolo --dataset-root <数据集根目录> --class-config configs/classes.yaml --output reports/...

# 第 5 步：生成训练配置文件
insightclass write-yolo-yaml --dataset-root <数据集根目录> --class-config configs/classes.yaml --output <输出路径>
```

```

                          ┌─────────────────────┘
                          ▼
                ┌──────────────┐     ┌──────────────┐     ┌──────────────┐
                │  加载模型    │────▶│  推理预测    │────▶│  可视化      │
```

```bash

insightclass compare-experiments --experiments-root experiments --output reports/experiment_summary.csv
```

## 四、核心模块详解

### 4.1 `cli.py` — 命令行入口

**职责**：接收用户输入的命令，分发到对应的处理模块。

**工作方式**：
- 使用 Python 标准库 `argparse` 定义 10 个子命令
- 每个子命令解析参数后，调用对应模块的功能
- 例如 `insightclass train --config xxx.yaml` 会：读取 YAML → 创建 TrainingConfig → 调用后端的 train 方法

**支持的子命令**：

| 子命令 | 功能 | 调用的核心模块 |
|--------|------|----------------|
| `create-manifest` | 创建数据集清单 | `data.manifest` |
| `extract-frames` | 视频抽帧 | `data.video_ops` |
| `inspect-yolo` | 检查标注质量 | `data.yolo` |
| `write-yolo-yaml` | 生成训练配置 | `data.yolo` |
| `train` | 训练模型 | `backends.factory` → `ultralytics_backend` |
| `validate` | 验证模型 | 同上 |
| `predict` | 推理预测 | 同上 |
| `render-first-frame` | 可视化第一帧 | `visualization.pipeline` |
| `compare-experiments` | 实验对比 | `evaluation.experiments` |
| `serve` | 启动 Web 服务 | `web.server` |

### 4.2 `schemas.py` — 核心数据结构

**职责**：定义整个系统中流转的数据格式，就像"数据的骨架"。

所有结构都用 `@dataclass(slots=True)` 定义（一种轻量级的类），每个都有 `to_dict()` 方法可以转成字典。

| 数据结构 | 用途 | 关键字段 |
|----------|------|----------|
| `DatasetManifest` | 描述一个数据集 | dataset_name, splits（哪些视频分到 train/val/test）, classes |
| `TrainingConfig` | 训练参数 | model_weights, epochs, batch_size, device, image_size |
| `InferenceConfig` | 推理参数 | weights_path, source, confidence, iou |
| `ExperimentRecord` | 一次训练的完整记录 | experiment_id, metrics, artifacts, hyperparameters |
| `DetectionRecord` | 单个检测框 | xyxy（坐标）, confidence（置信度）, class_id, class_name |
| `FramePrediction` | 一帧的所有检测结果 | frame_index, detections 列表 |

### 4.3 `backends/` — 模型后端（策略模式）

**职责**：封装所有与 AI 模型交互的逻辑。采用"策略模式"，可以轻松切换不同的模型框架。

**设计思路**：

```
base.py（抽象基类）      定义"后端必须做什么"
    │                    5 个抽象方法：train, validate, predict, load_predictions, export_artifacts
    ▼
factory.py（工厂）       根据名字找后端
    │                    build_backend("ultralytics") → UltralyticsBackend 实例
    ▼
ultralytics_backend.py   具体实现：调用 ultralytics 库的 YOLO 模型
```

**为什么要这样设计？**
如果以后想用别的模型框架（比如 detectron2、mmdetection），只需要写一个新的后端文件，在 factory.py 注册一下，其他代码完全不用改。

**`UltralyticsBackend` 的核心方法**：

| 方法 | 做什么 | 输入 | 输出 |
|------|--------|------|------|
| `train()` | 调用 YOLO 训练 | TrainingConfig | ExperimentRecord |
| `validate()` | 调用 YOLO 验证 | TrainingConfig | metrics 字典 |
| `predict_images_or_video()` | 调用 YOLO 推理 | InferenceConfig | 输出目录路径 |
| `load_predictions_as_sv_detections()` | 推理并返回结构化结果 | InferenceConfig | FramePrediction 列表 |
| `export_artifacts()` | 收集训练产物路径 | 实验目录 | artifact 路径字典 |

### 4.4 `data/` — 数据处理模块

**职责**：处理从原始视频到可训练数据集的全过程。

#### `manifest.py` — 数据集清单

**功能**：扫描视频目录 → 按比例随机划分 train/val/test → 生成清单 YAML

**关键设计**：视频级别划分（不是帧级别）。同一个视频的所有帧必须在同一个划分中，否则会导致数据泄露（模型在训练时看到了测试集的帧）。

```
输入：raw_videos/ 目录（里面有若干 .mp4 文件）
      │
      ▼
discover_videos()  ─── 递归扫描所有视频文件
      │
      ▼
build_split_map()  ─── 用固定种子随机打乱，按 7:2:1 划分
      │
      ▼
create_manifest()  ─── 组装成 DatasetManifest 对象
      │
      ▼
输出：manifest.yaml（记录每个划分里有哪些视频）
```

#### `video_ops.py` — 视频抽帧

**功能**：读取清单，逐个打开视频，按指定 FPS 截图保存。

**处理逻辑**：
1. 根据清单找到每个划分下的视频
2. 用 OpenCV 打开视频
3. 按 `fps` 参数计算帧间隔（比如 fps=1 就是每秒取 1 帧）
4. 如果设了 `max_frames_per_video`，每个视频最多取这么多帧
5. 如果设了 `target_width`，把图片缩放到指定宽度（保持比例）
6. 保存为 JPG，文件名格式：`{视频名}_f{帧号:06d}.jpg`
7. 写一个 `frame_index.csv` 记录每张图的来源和元数据

#### `yolo.py` — 标注质量检查

**功能**：扫描 YOLO 格式的标注文件，报告各种问题。

**检查项**：
- 有图片但没有对应标注文件
- 标注文件为空
- 类别 ID 超出范围
- 坐标超出 [0,1] 范围
- 检测框面积太小（可能是误标）
- 各类别的数量分布

### 4.5 `evaluation/` — 实验管理模块

**职责**：保存、加载、汇总训练实验的记录。

**核心功能**：
- `save_experiment_record()`：把一次训练的完整信息保存为 JSON
- `collect_experiment_records()`：扫描 experiments 目录，收集所有实验记录
- `export_experiment_summary()`：把所有实验的指标导出成 CSV，方便对比

**实验记录包含什么**：
- 实验 ID（目录名）
- 使用的后端和模型权重
- 数据集版本
- 类别名称
- 超参数（完整配置）
- 指标（mAP、precision、recall 等）
- 产物路径（权重文件、结果图等）

### 4.6 `visualization/` — 可视化模块

**职责**：在图片或视频帧上绘制检测框和标签。

**依赖**：`supervision` 库（一个专门做检测可视化的库）。

**工作流程**：
1. 读取图片/视频帧
2. 把 `FramePrediction` 转成 `supervision.Detections` 格式
3. 用 `BoxAnnotator` 画框
4. 用 `LabelAnnotator` 写标签（类别名 + 置信度）
5. 保存结果图片

### 4.7 `web/` — Web 前端模块

**职责**：提供浏览器端的实时检测界面。

#### `server.py` — FastAPI 服务端

**4 个 API 接口**：

| 接口 | 方法 | 作用 |
|------|------|------|
| `/` | GET | 返回网页（HTML 模板渲染） |
| `/api/experiments` | GET | 返回所有实验列表（JSON） |
| `/api/detect/frame` | POST | 上传一张图片，返回检测结果 |
| `/api/detect/upload` | POST | 上传一个视频，返回逐帧检测结果 |

**两种检测模式**：
- **摄像头模式**：浏览器用 `getUserMedia` 打开摄像头 → 每 200ms 抓一帧 → POST 到 `/api/detect/frame` → Canvas 叠加检测框
- **视频上传模式**：选择本地视频 → POST 到 `/api/detect/upload` → 服务端逐帧推理 → 返回全部结果 → 前端同步播放+叠加

#### `model_cache.py` — 模型缓存

**问题**：YOLO 模型加载很慢（几秒），不能每次请求都重新加载。

**方案**：用一个字典缓存已加载的模型，按权重路径索引。第一次请求时加载，后续直接用缓存。服务关闭时清理。

#### `app.js` — 前端交互逻辑

**核心逻辑**：
- 摄像头模式：用递归 `setTimeout` 控制检测频率（上一个请求返回后才发下一个，防止堆积）
- 视频模式：用 `requestAnimationFrame` 节流重绘（配合 `ontimeupdate` 事件）
- 检测框绘制：根据缩放比例计算坐标，在 Canvas 上画矩形和标签

### 4.8 `utils/` — 工具函数

| 文件 | 功能 |
|------|------|
| `paths.py` | `ensure_dir()` 创建目录；`is_video_file()` 判断是否视频文件 |
| `serialization.py` | `load_yaml()` / `save_yaml()` / `load_json()` / `save_json()` 读写文件 |

### 4.9 `optional.py` — 可选依赖检测

**职责**：优雅处理可选依赖缺失的情况。

**核心思想**：核心包只依赖 numpy + PyYAML + opencv-python。ultralytics、supervision、fastapi 等都是可选的，用到时才检测。

- `has_package("ultralytics")` → 返回 True/False
- `require_package("ultralytics", "训练功能")` → 如果没装就抛出友好错误信息

### 4.10 `scripts/` — 独立工具脚本

这些脚本不属于核心包，直接用 `python scripts/xxx.py` 运行。

| 脚本 | 功能 |
|------|------|
| `record_multi_rtsp.py` | 同时连接多路海康威视摄像头录制视频，支持子码流回退、主线程预览 |
| `rtsp_preview.py` | 单路摄像头预览+诊断（自动检测主码流/子码流哪个能用） |
| `sample_for_annotation.py` | 从训练集随机抽样图片，方便拿去标注 |
| `server_setup.sh` | Linux 服务器一键配置（创建 venv、安装依赖、生成数据集配置） |
| `server_train.sh` | Linux 服务器一键启动训练（支持 yolo11n/yolo26n，前台/后台模式） |

## 七、技术亮点

### 7.1 可插拔后端设计

通过抽象基类 + 工厂模式，新增模型框架只需要：
1. 继承 `DetectorBackend`
2. 实现 5 个方法
3. 在 `factory.py` 注册

数据处理、评估、可视化代码完全不用改。

### 7.2 可选依赖懒加载

核心包只依赖 3 个轻量库。重量级依赖（ultralytics、supervision、fastapi）在真正用到时才 import，没装也不会报错（只在调用相关功能时提示）。

### 7.3 视频级数据划分

划分在视频级别而不是帧级别，防止同一个视频的不同帧同时出现在训练集和测试集（数据泄露）。

### 7.4 完整的实验管理

每次训练自动保存完整的实验记录（超参 + 指标 + 产物路径），支持跨实验对比导出 CSV。
