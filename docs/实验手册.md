# 实验手册

## 1. baseline 命名规范

建议统一格式：

`{stage}_{backend}_{weights}_{dataVersion}_{imgsz}_{epochs}_{tag}`

示例：

- `baseline_ultralytics_yolo11n_v0_1_960_e80`
- `baseline_ultralytics_yolo11s_v0_1_960_e120_augv2`

## 2. 允许变化和必须固定的参数

### 必须固定

- 数据版本
- 类别表
- train / val / test 视频级划分
- 指标读取口径
- 评估集
- 随机种子

### 可以变化

- 模型权重
- `imgsz`
- `epochs`
- `batch`
- 数据增强策略
- 训练轮次和 patience

## 3. 训练强度分层

建议至少分三档：

- 轻量基线：
  - `yolo11n`
  - `imgsz=640/960`
  - `epochs=50-80`
- 中等强度：
  - `yolo11s`
  - `imgsz=960`
  - `epochs=100-150`
- 强化对比：
  - 相同模型
  - 提高 `epochs`
  - 有控制地增加数据增强

比较时一次只改 1 到 2 个变量，避免实验结论混乱。

## 4. 每次实验必须记录的内容

- 实验 ID
- 时间
- 模型和权重
- 数据版本
- 类别表
- 训练参数
- 核心指标
- 失败现象
- 主观观察
- 是否进入下一轮

## 5. 进入下一轮的判断标准

满足以下任意两项，可进入下一轮：

- `mAP50-95` 明显高于当前 baseline
- 关键类别 `phone_use` 召回提升明显
- 后排小目标误检下降
- 演示视频主观效果更稳定
- 推理速度仍在可接受范围

## 6. 首期推荐实验序列

1. `yolo11n` 小模型打通全流程
2. 固定数据和参数，试 `yolo11s`
3. 固定模型，增加 `epochs`
4. 对 `phone_use` 小目标场景调 `imgsz`
5. 复盘误检和漏检，再决定是否进入二阶段方案
